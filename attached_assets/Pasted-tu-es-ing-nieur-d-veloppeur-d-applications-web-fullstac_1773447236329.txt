PROJET : PLATEFORME WEB FINANCIÈRE & BLOCKCHAIN D
1. PRÉSENTATION GÉNÉRALE DU PROJET
Le présent document constitue une étude de faisabilité détaillée pour la
Conception et le développement d’une plateforme web financière inspirée du
modèle du site bofblockchain.com, intégrant des fonctionnalités avancées de
gestion de comptes clients, de soldes multidevises et de transactions financières
supervisées. La plateforme sera accessible via navigateur web et destinée à des
utilisateurs finaux souhaitant gérer leurs comptes, consulter leurs soldes et
effectuer des demandes de transactions financières de manière sécurisée.
2. DESCRIPTION DU BESOIN
Le client souhaite mettre en place une solution web permettant :
L’enregistrement des utilisateurs via un formulaire sécurisé
L’activation manuelle des comptes par un administrateur
La gestion d’un espace client personnalisé
La consultation de soldes en plusieurs devises (EUR, USD, BTC)
L’ajout de coordonnées bancaires
La demande de virements bancaires
La demande de transferts de cryptomonnaies
Le suivi en temps réel de l’état des transactions
L’administration complète de la plateforme via un tableau de bord
sécurisé
3. ANALYSE TECHNIQUE DU PROJET
3.1 Nature du projet
Le projet est une application web dynamique et non un simple site vitrine.
3.2 Technologies utilisées
Frontend : Réact & Bootstrap (mise en forme et responsive design)
JavaScript (interactions utilisateur, validations, affichage dynamique),
AXIOS (communications API)
Backend : Node.js/Express.js, API Rest sécurisée, authentification JWT
Base de données : postgreSQL
3.3 Répartition Frontend / Backend
JavaScript côté frontend sera utilisé uniquement pour l’interface
utilisateur
Toutes les opérations sensibles (authentification, gestion des soldes,
transactions, permissions) seront gérées côté backend en Nodejs/Expressjs pour
garantir la sécurité.
4. FONCTIONNALITÉS DÉTAILLÉES
4.1 Fonctionnalités côté client
Création de compte utilisateur
Connexion sécurisée
Activation du compte par l’administrateur
Accès à un tableau de bord personnel
Consultation du solde disponible (EUR, USD, BTC)
Ajout et gestion des coordonnées bancaires
Soumission de demandes de virements bancaires
Soumission de demandes de transferts en cryptomonnaies
Suivi de l’état des transactions (en attente, en cours, validé, rejeté)
Réception de messages automatiques de notification
4.2 Fonctionnalités côté administrateur
Accès à un dashboard sécurisé réservé à l’administrateur
Gestion des utilisateurs (activation, suspension, suppression)
Gestion et ajustement des soldes clients
Validation ou rejet des transactions
Consultation de l’historique complet des opérations Journalisation (logs)
des actions administratives
5. SÉCURITÉ, LIMITES ET RESPONSABILITÉS
5.1 Sécurité
Hashage sécurisé des mots de passe
Protection contre les injections SQL Sécurisation des sessions utilisateur
Accès restreint aux interfaces sensibles
Utilisation obligatoire du protocole HTTPS
5.2 Limites légales
La plateforme ne constitue pas une banque officielle
Elle ne remplace pas un établissement financier agréé
Les transactions peuvent être : simulées (gestion interne) ou soumises à
validation manuelle Toute intégration d’API bancaires ou crypto
officielles fera l’objet d’un développement et d’un coût supplémentaire.
